## Architecture notes

**Routing:** wouter SPA. Every route also has a prerendered `.html` file under `dist/public/` (built by `scripts/prerender.ts` post-Vite). Cloudflare serves static files first; the SPA fallback handles client-side routing for everything else.

**Metadata:** `src/lib/meta.ts` is the single source of truth for all 15 routes' titles, descriptions and OG data. The client hook (`use-page-meta.ts`) applies it on SPA transitions; the prerender script injects it into static HTML at build time. Do not add per-page metadata anywhere else.

**Contact form / enquiry endpoint:**
- Front-end: `src/pages/Contact.tsx` -- POSTs JSON to `/api/enquiry`; prefills from `?type=` param
- Worker: `worker/index.ts` -- validates, rate-limits, sends via `send_email` binding
- Config: `wrangler.toml` -- `send_email`, `ratelimits`, `vars` (CONTACT_INBOX, ENQUIRY_FROM)
- The binding is currently inactive (requires dashboard setup -- see `docs/email-migration.md`)
- The `mailto:` fallback stays visible until the live endpoint is verified

**Project data:** `src/data/projects.ts` -- 10 projects, each with a `layout` field (`"editorial" | "identity" | "campaign"`). This drives `VARIANT_TEXT` in `ProjectDetail.tsx` and the `variant` prop on `ProjectContentFlow.tsx`.

**Spread compositing:** `DoubleSpread.tsx` composites left/right page exports at render time. Never pre-merge spreads as flat files. SPREAD_ASPECT_RATIOS has entries for all projects that use DoubleSpread. Identity/packaging projects use the "plates" variant (standalone boards, no spread blocks).

**Work-page scroll restoration:** `use-work-return.ts` -- saves scroll offset + anchor to sessionStorage on card click; restores on browser Back, "Back to Work" link, and reload. `history.scrollRestoration = "manual"` is set globally. The link state `{ fromWork: true }` triggers `history.back()` instead of a navigate; `{ restoreWork: true }` triggers sessionStorage restore.
